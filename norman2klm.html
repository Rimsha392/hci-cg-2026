Clean Comparison (Class & Exam Reference)

Model-based evaluation: what each model explains vs. predicts
Model Category What it Explains What it Predicts
Norman Cognitive / descriptive Understanding, errors, feedback, gulfs ❌ Nothing (no performance prediction)
GOMS Predictive / analytic Task structure for expert users ✔ Relative efficiency
KLM Predictive / quantitative Low-level physical and motor actions ✔ Execution time

KLM Operators (Execution-Level Primitives)

KLM decomposes the execution phase of interaction into primitive operators. The user is assumed expert and the method already chosen.

KLM operators (Dix et al., Chapter 12)
Operator Name Meaning Example
KKeystrokingPressing a key (including modifiers)Typing a character or Delete
BButton pressPressing a mouse buttonLeft-click
PPointingMoving pointer to a targetMove cursor to icon
HHomingSwitch hand between devicesKeyboard → mouse
DDrawingDrawing lines with pointerDrag to draw shape
MMental prepPreparing for physical actionPause before clicking control
RSystem responseWaiting for system feedbackApplication loading

Worked Example: Correcting a Single Character

User notices a typo, deletes it, retypes it, and returns to insertion point.

  1. Move hand to mouse
  2. Position mouse after error
  3. Return to keyboard
  4. Delete character
  5. Type correction
  6. Reposition insertion point

Example KLM sequence:

H[mouse] P B[LEFT] H[keyboard] M K[DELETE] K[char] H[mouse] M P B[LEFT]

Key Takeaways

KLM consists of 7 primitive operators (K, B, P, H, D, M, R) used to predict execution time for expert, error-free interaction.

KLM models execution, not planning or interpretation.
